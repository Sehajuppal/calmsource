# Break-the-App QA Methodology

This document tracks our methodology for Mission 21 chaos testing. 

## Approach
We use adversarial sub-agents to stress-test the application beyond happy paths. 

## Focus Areas
- SA1: Build Stability
- SA2: Bad Source Inputs
- SA3: IPTV Chaos
- SA4: Addon Chaos
- SA5: Playback & Fallback
- SA6: Search & Ranking
- SA7: Mobile UI Abuse
- SA8: Android TV Abuse
- SA9: Privacy & Persistence
- SA10: Performance

## Test Fixtures
All agents are strictly instructed to use sanitized, mock, or legal sources. No piracy addons or real secret endpoints are to be used or logged.
